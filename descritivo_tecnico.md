1. Backend & Data Engineering (O Core)Objetivo: Criar a API de ingestão e o motor de scoring.Ingestion Pipeline: Precisamos de um worker que consuma a API Sonar (Perplexity). O prompt de sistema deve forçar um output JSON estruturado contendo: resumo, tags_entidade, sentimento e a pergunta_validacao.Vector Database (Memory): Implementar Supabase (pgvector). Toda notícia nova deve gerar um embedding. Antes de chamar a Perplexity, o sistema deve checar similaridade no banco para evitar gastos desnecessários com tokens (Cache Semântico).Algoritmo de Scoring ($S$): Criar a tabela user_affinity. O cálculo de peso deve ser incremental.Click = +5; Time_on_page > 30s = +10; Positive_Question_Response = +50.Implementar um Cron Job para rodar o decaimento temporal (reduzir pesos inativos em 5% a cada 24h).2. Frontend & UX (Interface Orgânica)Objetivo: Criar uma UI que pareça viva e que não gere fadiga visual.The Glass Isle Navbar: O input de busca deve ser o herói. Ele nasce no centro da Home e, ao scrollar, sofre uma transição suave para a navbar (Glassmorphism). Quero 0ms de percepção de lag.Widget-Based Layout: O grid da home deve ser modular. Utilizar CSS Grid/Flexbox permitindo que os blocos de categoria sejam reordenados conforme o user_affinity score.Micro-interações de Feedback: O card de notícia deve ter um estado "Question". Se o usuário não tem a tag daquela notícia confirmada, o card expande um pequeno bubble com a pergunta gerada pela IA.3. Design System (Identidade e Experiência)Objetivo: Sair do "feio" para o "Premium Minimalista".Paleta de Cores: Focar em tons Pastel Low-Contrast (fundo #F5F5F5 ou similar) para reduzir o cansaço ocular. A tipografia deve ser robusta para leitura longa (Serifada nos corpos de texto, Sans-serif nos títulos).Anatomia do Card: 1.  Imagem de destaque (opcional/AI generated).2.  Headline.3.  Chips de categorias (vão sumindo conforme a confiança do algoritmo aumenta).4.  Slot de Interação (onde entra a pergunta da IA).4. Product & Monetização (Business Logic)Objetivo: Implementar as travas de valor.Feature Text2Speech (TTS): Integrar ElevenLabs ou OpenAI Whisper/TTS. O backend deve pré-processar o áudio apenas sob demanda para economizar.Rate Limiting: Implementar middleware de controle de requisições. Contas free batem no cache; contas premium disparam queries frescas na Perplexity.Persona Switch: No perfil do usuário, criar um toggle de "Tom da IA". Isso altera o prompt enviado para a camada de síntese (System Prompt: "Você é um comentarista ácido/técnico/amigável").Definição de Pronto (DoD)A task só será considerada entregue quando:A pesquisa de um usuário retornar um resumo em < 3 segundos.A resposta de uma "Pergunta de Validação" alterar o peso da categoria no DB em tempo real.O layout se ajustar dinamicamente aos pesos de interesse sem quebrar o grid.
